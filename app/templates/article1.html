    What is it?

    Pylint is a source-code, bug and quality checker for the Python programming language. It is named following a common convention in Python of a "py" prefix, and a nod to the C programming lint program. It follows the style recommended by PEP 8, the Python style guide and can:
  • Check the length of each line
  • Check that variable names are well-formed according to the project's coding standard
  • Check that declared interfaces are truly implemented
  • Try this command in Pycharm to run the Pylint test:

    • pytest --pylint --cov

    Tips & Tricks:

  • Disable Pylint Errors - sometimes Pylint is gives an error that can be addressed by adding code to disable pylint errors. When testing the Calculation class, I got an error from Pylint stating too-few-public-methods. This error is caused by the need to add more methods. This error can be disabled by adding a comment with a specific Pylint code immediately after defining the class.
    • @staticmethod #this is an example of a calling method def divide_numbers(value_a, value_b): """division two numbers and store the result""" # create a division object using the factory we created on the calculation class division = Division.create(value_a, value_b) Calculator.add_calculation_to_history(division) return Calculator.get_results_of_last_calculation_added_to_history()
  • Missing Final Newline - the missing-final-newline error means that there should be an additional newline character at the end of the file. The fix is to add an empty new line by hitting ENTER at the end of your file.
    • C: 1, 0: Final newline missing (missing-final-newline) C: 2, 0: Final newline missing (missing-final-newline) C: 3, 0: Final newline missing (missing-final-newline) C: 4, 0: Final newline missing (missing-final-newline) C: 5, 0: Final newline missing (missing-final-newline)
  • Missing Docstring - The missing docstring error occurs if you forget to add a docstring to the beginning of your file. To correct the error, add a docstring in the corresponding line specified in the error.
    • """This is our calculation base class & abstract class""" class Calculation: # pylint: disable=too-few-public-methods """constructor & it is the 1st function called when an object of the class is instantiated""" def __init__(self, value_a, value_b): # pylint: disable=too-few-public-methods """class factory method""" self.value_a = value_a self.value_b = value_b @classmethod def create(cls, value_a, value_b): """class method""" return cls(value_a, value_b)
