  • Class - a user-defined prototype for an object that defines a set of attributes that characterize any object of the class. The attributes are data members (class variables and instance variables) and methods, accessed via dot notation.
  • Class Method - a method that is bound to the class and not to the object of the class. It takes a class parameter that points to the actual class and not the object instance. They are generally used to create factory methods.
  • Class Variable - a variable that is shared by all instances of a class. Class variables are defined within a class but outside any of the class's methods. Class variables are not used as frequently as instance variables are.
  • Constructor - the creation of an instance of a class.
  • Data Member - a class variable or instance variable that holds data associated with a class and its objects.
  • Design Patterns - a common way of solving well known problems. They represent the best ways to structure your code to achieve the desired outcome, particularly for creating and defining classes, methods, objects, etc.
  • Facade - a design pattern that allows an interface or class to create an object but lets subclasses decide which class or object to instantiate. It allows objects to be created without exposing the logic to the user, and allows the user to use the same common interface each time.
  • Factory - is an object for creating other objects. A subroutine that returns a "new" object may be referred to as a "factory", as in factory method or factory function. This is a basic concept in OOP, and forms the basis for a number of related software design patterns.
  • Fixture - provides a defined and consistent context for each test. In Pytest, they are functions that we define for this purpose.
  • Function Overloading - the assignment of more than one behavior to a particular function. The operation performed varies by the types of objects or arguments involved.
  • Instance Variable - a variable that is defined inside a method and belongs only to the current instance of a class.
  • Inheritance - the transfer of the characteristics of a class to other classes that are derived from it.
  • Instance - An individual object of a certain class. An object obj that belongs to a class Circle, for example, is an instance of the class Circle.
  • Instantiation - the creation of an instance of a class.
  • Method - a special kind of function that is defined in a class definition.
  • Namespace - an object-oriented concept which allows for organization of variables, functions, and other elements of a program. It encapsulates code such that to access that code, you must first reference the namespace to which you are referring. Programmers can create other namespaces, thus organizing code and reducing pollution of the global namespace.
  • Object - a unique instance of a data structure that's defined by its class. An object comprises both data members (class variables and instance variables) and methods.
  • Object-Oriented Programming - a programming paradigm based on the concept of "objects", which can contain data and code: data in the form of fields, and code, in the form of procedures.
  • Operator Overloading - the assignment of more than one function to a particular operator.
  • Property - the creation of an instance of a class.
  • Static Method - a method that is bound to the class and not to the object of the class. A static method cannot access or modify the state of a class, unlike a class method, and are generally used to create utility functions for a class.
  • Type Cast - involves the act of changing one data type into another. For example, converting an "int" into a "float".
  • Type Hint - an annotation that specifies the expected type for a variable, a class attribute, or a function parameter. Type hints are optional and not enforced by Python but can be very useful.
  • Unit Test - a software testing method by which individual units of source code are put under various tests to determine whether they perform as intended.
